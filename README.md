# サンプルキット: ランダム問題生成 × 自動採点

これは、リポジトリ直下の `仕様書.md` で説明している仕組みを**実際に動く最小構成**にしたサンプルです。
「梁のたわみ計算」というダミー科目を題材に、次の一連の流れを手元で試せます。

```
問題定義（weekN.py） → PDF生成（演習・小テスト・再テスト） → スキャン → AI採点 → 集計
```

他の科目向けに作り直すときは、このフォルダをコピーして `SampleCourse/` 以下の
`weekN.py`（問題定義）だけを自分の科目に書き換えるのが基本的な流れです
（`packages/` 以下の共通エンジンはそのまま使い回せます）。

## 実物（本物）とダミーの区別

| 部分 | 本物？ | 備考 |
|---|---|---|
| `packages/quiz/` の問題生成・PDF生成・OMR補正・採点ロジック | **本物** | 元リポジトリからそのままコピー |
| `packages/models/*.pth` | **本物**（一部） | 実際に手書き解答で学習した重み。全アンサンブル（本番は数十個）から代表10個を抜粋 |
| `SampleCourse/` の問題（梁のたわみ計算） | 新規サンプル | 実演用に新規作成。ランダマイズの実演が目的 |
| `output/` への出力先 | ダミー | 元は OneDrive/SharePoint 同期フォルダだったが、ローカルの `output/` フォルダに単純化 |
| 成績簿（Excel COM）転記・SharePoint配布 | **含まれない** | 学校固有の運用に強く依存するため意図的に省略（`仕様書.md` 参照） |
| `scan_samples/` の解答用紙スキャン画像 | **本物** | 実際に印刷・記入・スキャンしたものを同梱済み |

## 必要な環境

- [pixi](https://pixi.sh/)
- **LaTeX 環境**（MiKTeX（Windows）や TeX Live（Mac/Linux）等）
  数式の解説（例: たわみの計算過程）を `matplotlib` の `usetex` 経由でレンダリングするため必須。
  未インストールだと解説図の生成でエラーになる（問題文・採点自体は動く）。
- 日本語フォント（Windows なら標準搭載の Meiryo で足りる。詳細は `packages/quiz/fonts/README.md`）

## セットアップ

```bash
cd sample_kit
pixi install
```

`SampleCourse/2026/config.xlsx` は同梱済み（`SampleCourse/make_config.py` で生成したもの）。
週番号・使う `weekN.py`・実施日（＝乱数シード）を管理している。Excel で直接開いて日付を
変えれば、次に生成する小テストの問題が変わる。

## 使い方（GUI）

```bash
pixi run python sample_gui.py
```

### 週次小テストタブ
1. 科目・週を選ぶ（このサンプルには week1〜3 の3パターンがある）。
2. 「演習PDF生成」: マーカー・解答欄なし、模範解答つきの配布用プリントを生成。
3. 「小テストPDF生成」: マーカー・解答欄あり。シードは `config.xlsx` の実施日から自動導出。
   **同じ week でも config.xlsx の日付を変えるだけで別問題になる。**
4. 印刷して手書き記入 → スキャン/撮影。
   画像は `output/.../minitest/1st/weekNN/scan/` に置く。
5. 「採点」: スキャン画像を読み取り、自動生成した正答と照合して採点する。
   出席番号の重複・欠番などの診断もログに表示される。
6. 「返却PDF生成」: 採点結果を書き込んだPDFを生成する。

### 個別再テストタブ
出席番号と受験回数を指定して「再テストPDF生成」を押すと、**その学生専用のシード**
（`出席番号2桁+受験回数2桁`）で問題を再生成する。同じ学生の1回目・2回目でも、
問題の構造は同じだが数値・条件は別物になることを確認できる。

再テストPDFには氏名欄に出席番号を事前印字したうえで、**出席番号・週・受験回のシード・
ページ数を埋め込んだQRコード**も印字される（元リポジトリ Grading と同じ仕組み）。
採点方法は2通り:
- 「再テスト採点」: 従来どおり出席番号・受験回数を指定し、`retry/{番号}_{回数}/scan/`
  に置いたスキャンを採点する。
- 「QR一括採点（フォルダ自動振分）」: 複数人・複数週の再テストのスキャンを、学生ごとに
  仕分けずに `output/{科目}/{年度}/minitest/retry_scan/` へまとめて置くだけで、QRコード
  から出席番号・週・受験回を自動判定して採点する。複数ページの答案の結合や、
  同じ答案を誤って重複スキャンした場合の除外もQRメタデータをもとに自動で行われる。

採点中は「採点結果プレビュー」「AI入力プレビュー（白黒）」に、その都度読み取った答案・
AI入力画像が表示される（採点のたびに `files/result/image/` へ画像を書き出す旧来のデバッグ
出力は廃止し、GUI上でのデモ表示に置き換えた）。
採点後は「返却PDF生成」で、その学生・週・受験回の `retry/{番号}_{回数}/result/` に
解説PDF・個人の返却PDFを生成する（「再テスト採点」「QR一括採点」どちらの結果でも使える）。

## AI認識モデルについて

`packages/models/` には実際に手書き数字で学習した CNN の重み（`.pth`）を、数字認識・符号認識
それぞれ5個ずつ（本番はもっと多くのモデルをアンサンブルしている）同梱している。合成データではなく
本物の重みなので、`scan_samples/` の同梱スキャン画像で本当に推論が動く。
モデルの選定・比較・代替候補（クラウドOCR・LLM併用など）については `仕様書.md` の
「4. 手書き数字・記号認識モデルの選択肢」を参照。

## フォルダ構成

```
sample_kit/
  packages/
    quiz/            共通エンジン（問題定義基底クラス・梁の力学計算・PDF生成・OMR補正など）
    AI.py            CNNモデルの定義
    Marksheet2.py    旧式マークシート読み取り（本サンプルでは未使用、参考として同梱）
    models/          学習済みモデルの重み（本物・抜粋）
  SampleCourse/
    2026/
      config.xlsx    週スケジュール（実施日=シード）
      1st/
        week1.py     片持ち梁+集中荷重
        week2.py     単純支持梁+分布荷重
        week3.py     張り出し梁+集中荷重（支点位置もランダム）
  sample_gui.py      GUI 本体
  scan_samples/      実スキャン画像（同梱済み）
  output/            生成物の出力先（.gitignore 済み）
```
