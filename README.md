# サンプルキット: ランダム問題生成 × 自動採点

これは、リポジトリ直下の `仕様書.md` で説明している仕組みを**実際に動く最小構成**にしたサンプルです。
「梁のたわみ計算」というダミー科目を題材に、次の一連の流れを手元で試せます。

```
問題定義（weekN.py） → PDF生成（演習・小テスト・再テスト） → スキャン → AI採点 → 集計
```

他の科目向けに作り直すときは、このフォルダをコピーして `SampleCourse/` 以下の
`weekN.py`（問題定義）だけを自分の科目に書き換えるのが基本的な流れです
（`packages/` 以下の共通エンジンはそのまま使い回せます）。

## 実物（本物）とダミーの区別

| 部分 | 本物？ | 備考 |
|---|---|---|
| `packages/quiz/` の問題生成・PDF生成・OMR補正・採点ロジック | **本物** | 元リポジトリからそのままコピー |
| `packages/models/*.pth` | **本物**（一部） | 実際に手書き解答で学習した重み。全アンサンブル（本番は数十個）から代表10個を抜粋 |
| `SampleCourse/` の問題（梁のたわみ計算） | 新規サンプル | 実演用に新規作成。ランダマイズの実演が目的 |
| `output/` への出力先 | ダミー | 元は OneDrive/SharePoint 同期フォルダだったが、ローカルの `output/` フォルダに単純化 |
| 成績簿（Excel COM）転記・SharePoint配布 | **含まれない** | 学校固有の運用に強く依存するため意図的に省略（`仕様書.md` 参照） |
| `scan_samples/` の解答用紙スキャン画像 | **本物** | 実際に印刷・記入・スキャンしたものを同梱済み |

## セットアップ（プログラミング未経験の方向け）

Windows パソコンを想定した手順。所要時間の目安は30分〜1時間（LaTeXのダウンロードに時間がかかる場合あり）。
コマンドは黒い（または青い）画面に文字を打ち込んで実行する「PowerShell」というアプリを使う。
コピー&ペーストで進められるので、表示されたコマンドをそのまま貼り付けて Enter を押せばよい。

### ステップ1: このリポジトリを手元に用意する

**Gitを使ったことがない方（ZIPダウンロード）**

1. ブラウザで https://github.com/yusaku-m/MinitestSampleKit を開く。
2. 緑色の「Code」ボタン→「Download ZIP」をクリック。
3. ダウンロードした zip ファイルを右クリック→「すべて展開」を選び、デスクトップなど
   好きな場所に展開する。`MinitestSampleKit` フォルダができる。

**Gitが使える方**

```powershell
git clone https://github.com/yusaku-m/MinitestSampleKit.git
```

### ステップ2: フォルダを開いた状態で PowerShell を起動する

エクスプローラーで `MinitestSampleKit` フォルダを開き、フォルダの中の何もない場所を
**Shift + 右クリック**し、「PowerShell ウィンドウをここに開く」（Windows 11 では
「ターミナルで開く」）を選ぶ。これ以降のコマンドはすべてこの画面に打ち込む。

（この操作を忘れて別の場所で PowerShell を開いてしまった場合は、`cd` の後に
フォルダのパスを貼り付ければ移動できる。例: `cd C:\Users\名前\Desktop\MinitestSampleKit`）

### ステップ3: pixi をインストールする

pixi は、Pythonや必要なライブラリを自動で用意してくれる管理ツール。次のコマンドを
コピーして貼り付け、Enter を押す。

```powershell
powershell -ExecutionPolicy Bypass -c "irm -useb https://pixi.sh/install.ps1 | iex"
```

数十秒でインストールが終わる。**終わったら、開いている PowerShell を一度閉じて、
ステップ2の要領で開き直す**（インストールしたことをこのアプリに認識させるため）。

開き直したら、うまく入ったか次のコマンドで確認する。

```powershell
pixi --version
```

`pixi 0.xx.x` のようにバージョン番号が表示されれば成功。
（`'pixi' は、内部コマンドまたは外部コマンド...として認識されていません` と出る場合は、
PowerShell を閉じて開き直せているか確認する）

### ステップ4: LaTeX（MiKTeX）をインストールする

数式の解説図（たわみの計算過程など）を作るのに使う。無くても問題文の生成・採点自体は
動くが、あとで解説図のところだけエラーになるので、先に入れておくのがおすすめ。

1. https://miktex.org/download を開く。
2. 「Basic MiKTeX Installer」の 64-bit 版をダウンロードして実行する。
3. インストーラの指示にしたがって進める（基本はすべて既定値のままで問題ない）。
   「Install missing packages on-the-fly」（未インストールのパッケージを自動で
   追加するか）を聞かれたら「Yes」を選んでおくと、あとで数式を使うたびに
   手動でパッケージを探す手間がなくなる。

日本語フォントは Windows 標準の Meiryo で足りるため、追加の作業は不要
（詳細は `packages/quiz/fonts/README.md`）。

### ステップ5: 必要なライブラリをまとめてインストールする

ステップ2の要領で、`MinitestSampleKit` フォルダで PowerShell を開き（すでに開いている
場合はそのままでよい）、次のコマンドを実行する。

```powershell
pixi install
```

Python本体や、AI認識に使う PyTorch・OpenCV などのライブラリが自動でダウンロードされる。
初回は数分〜十数分かかることがある（進捗が表示されるので、止まっているように見えても
そのまま待つ）。

### ステップ6: GUIを起動する

同じ画面で次を実行する。

```powershell
pixi run python sample_gui.py
```

「ランダム問題生成 × 自動採点 サンプルGUI」というウィンドウが立ち上がれば成功。
以降の操作は下の「使い方（GUI）」を参照。

### うまくいかないときは

- `pixi : 用語 'pixi' は...認識されません` → ステップ3のあとに PowerShell を
  閉じて開き直したか確認する。
- 解説図の生成でLaTeX関連のエラーが出る → ステップ4のMiKTeXが入っているか確認する。
  MiKTeXは初回、数式に必要なパッケージをその都度自動ダウンロードするため、
  ネット接続と数分の待ち時間が必要になることがある。
- 上記以外のエラーが出た場合は、画面に表示された文字（特に赤字の部分）をそのまま
  コピーして、詳しい人に質問するとよい。

`SampleCourse/2026/config.xlsx` は同梱済み（`SampleCourse/make_config.py` で生成したもの）。
週番号・使う `weekN.py`・実施日（＝乱数シード）を管理している。Excel で直接開いて日付を
変えれば、次に生成する小テストの問題が変わる。

## 使い方（GUI）

```bash
pixi run python sample_gui.py
```

### 週次小テストタブ
1. 科目・週を選ぶ（このサンプルには week1〜3 の3パターンがある）。
2. 「演習PDF生成」: マーカー・解答欄なし、模範解答つきの配布用プリントを生成。
3. 「小テストPDF生成」: マーカー・解答欄あり。シードは `config.xlsx` の実施日から自動導出。
   **同じ week でも config.xlsx の日付を変えるだけで別問題になる。**
4. 印刷して手書き記入 → スキャン/撮影。
   画像は `output/.../minitest/1st/weekNN/scan/` に置く。
5. 「採点」: スキャン画像を読み取り、自動生成した正答と照合して採点する。
   出席番号の重複・欠番などの診断もログに表示される。
6. 「返却PDF生成」: 採点結果を書き込んだPDFを生成する。

### 個別再テストタブ
出席番号と受験回数を指定して「再テストPDF生成」を押すと、**その学生専用のシード**
（`出席番号2桁+受験回数2桁`）で問題を再生成する。同じ学生の1回目・2回目でも、
問題の構造は同じだが数値・条件は別物になることを確認できる。

再テストPDFには氏名欄に出席番号を事前印字したうえで、**出席番号・週・受験回のシード・
ページ数を埋め込んだQRコード**も印字される（元リポジトリ Grading と同じ仕組み）。
採点方法は2通り:
- 「再テスト採点」: 従来どおり出席番号・受験回数を指定し、`retry/{番号}_{回数}/scan/`
  に置いたスキャンを採点する。
- 「QR一括採点（フォルダ自動振分）」: 複数人・複数週の再テストのスキャンを、学生ごとに
  仕分けずに `output/{科目}/{年度}/minitest/retry_scan/` へまとめて置くだけで、QRコード
  から出席番号・週・受験回を自動判定して採点する。複数ページの答案の結合や、
  同じ答案を誤って重複スキャンした場合の除外もQRメタデータをもとに自動で行われる。

採点中は「採点結果プレビュー」「AI入力プレビュー（白黒）」に、その都度読み取った答案・
AI入力画像が表示される（採点のたびに `files/result/image/` へ画像を書き出す旧来のデバッグ
出力は廃止し、GUI上でのデモ表示に置き換えた）。
採点後は「返却PDF生成」で、その学生・週・受験回の `retry/{番号}_{回数}/result/` に
解説PDF・個人の返却PDFを生成する（「再テスト採点」「QR一括採点」どちらの結果でも使える）。

## AI認識モデルについて

`packages/models/` には実際に手書き数字で学習した CNN の重み（`.pth`）を、数字認識・符号認識
それぞれ5個ずつ（本番はもっと多くのモデルをアンサンブルしている）同梱している。合成データではなく
本物の重みなので、`scan_samples/` の同梱スキャン画像で本当に推論が動く。
モデルの選定・比較・代替候補（クラウドOCR・LLM併用など）については `仕様書.md` の
「4. 手書き数字・記号認識モデルの選択肢」を参照。

## フォルダ構成

```
sample_kit/
  packages/
    quiz/            共通エンジン（問題定義基底クラス・梁の力学計算・PDF生成・OMR補正など）
    AI.py            CNNモデルの定義
    Marksheet2.py    旧式マークシート読み取り（本サンプルでは未使用、参考として同梱）
    models/          学習済みモデルの重み（本物・抜粋）
  SampleCourse/
    2026/
      config.xlsx    週スケジュール（実施日=シード）
      1st/
        week1.py     片持ち梁+集中荷重
        week2.py     単純支持梁+分布荷重
        week3.py     張り出し梁+集中荷重（支点位置もランダム）
  sample_gui.py      GUI 本体
  scan_samples/      実スキャン画像（同梱済み）
  output/            生成物の出力先（.gitignore 済み）
```
